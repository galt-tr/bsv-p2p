## Claude Code + PTY Bug
- **Root cause:** Claude Code crashes with `pty:true` on this machine. Works fine without PTY (`-p` flag).
- The error is in google-auth-library bundled inside Claude Code's CLI (TypeError on `.prototype`)
- **Workaround:** Use `claude -p` (non-interactive print mode) instead of `claude --dangerously-skip-permissions` with PTY
- **Lesson:** The coding-agent skill says to always use `pty:true` but that's wrong for this environment
